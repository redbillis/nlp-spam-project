# SMS Spam Detection NLP Pipeline

An end-to-end Natural Language Processing (NLP) project to classify SMS messages as "Spam" or "Ham" using the classic `spam.csv` dataset. This project was developed as part of a data analysis assessment to demonstrate a full NLP development lifecycle on a Linux environment.

## 📂 Project Structure

```text
.
├── data/
│   ├── spam.csv            # Raw dataset (Input)
│   └── spam_cleaned.csv    # Processed data (Generated by Notebook 01)
├── notebooks/
│   ├── 01_data_exploration.ipynb  # Wrangling, EDA, and Pre-processing
│   └── 02_model_training.ipynb     # TF-IDF, NB & SVM Modeling, Evaluation
├── requirements.txt        # Python dependencies
└── README.md               # Setup and documentation
```

## 🛠️ Prerequisites

- OS: Ubuntu (Local development machine)

- Python: `3.10+` and `venv` support

- Tools: `curl`, `unzip`

## 🚀 Quick Setup

1. Create and activate a virtual environment:

```
python3 -m venv venv
source venv/bin/activate
```

2. Install Python dependencies:

```
pip install -r requirements.txt
```

3. NLTK Data Setup (Stopwords & Punkt):

The notebooks utilize NLTK corpora. On some systems, the NLTK downloader may attempt IPv6 first and stall. Run these commands from the project root to manually install the corpora into the virtualenv using IPv4:

```
# Create directory and download zip files
mkdir -p venv/nltk_data
curl -4 -L -o venv/nltk_data/stopwords.zip [https://raw.githubusercontent.com/nltk/nltk_data/gh-pages/packages/corpora/stopwords.zip](https://raw.githubusercontent.com/nltk/nltk_data/gh-pages/packages/corpora/stopwords.zip)
curl -4 -L -o venv/nltk_data/punkt.zip [https://raw.githubusercontent.com/nltk/nltk_data/gh-pages/packages/tokenizers/punkt.zip](https://raw.githubusercontent.com/nltk/nltk_data/gh-pages/packages/tokenizers/punkt.zip)

# Unzip and cleanup
unzip -o venv/nltk_data/stopwords.zip -d venv/nltk_data
unzip -o venv/nltk_data/punkt.zip -d venv/nltk_data
rm venv/nltk_data/*.zip

# Organize folders for NLTK expectations
mkdir -p venv/nltk_data/corpora venv/nltk_data/tokenizers
mv -n venv/nltk_data/stopwords venv/nltk_data/corpora/stopwords || true
mv -n venv/nltk_data/punkt venv/nltk_data/tokenizers/punkt || true
chmod -R a+rX venv/nltk_data
```

## 🧪 Verifying Environment

Run the following command to verify that NLTK correctly locates the resources within your local venv:

```
python3 - <<'PY'
import nltk
import os
os.environ['NLTK_DATA'] = os.path.abspath('venv/nltk_data')
for p in ('corpora/stopwords','tokenizers/punkt'):
    try:
        print(p, 'FOUND at', nltk.data.find(p))
    except Exception as e:
        print(p, 'MISSING ->', e)
PY
```

## 📈 Running the Notebooks

To maintain data integrity and ensure the pipeline flows correctly, please run the notebooks in the following order:

1. Step 1: Exploration & Cleaning
   Open and run `notebooks/01_data_exploration.ipynb`. This notebook performs initial data wrangling, handles encoding issues, and performs text standardization. It exports the result to data/spam_cleaned.csv.

2. Step 2: Model Training & Comparison
   Open and run `notebooks/02_model_training.ipynb`. This notebook performs TF-IDF vectorization and compares the performance of Multinomial Naive Bayes against a Support Vector Machine (SVM).

Launch Jupyter Lab to begin:

```
jupyter lab
```

## 💡 Notes & Troubleshooting

- IPv6 Issues: If downloads are slow or hanging during the setup, ensure you use the `curl -4` flag provided in the instructions.

- Pathing: The notebooks utilize relative paths (`../data/`). Ensure the project structure remains intact for the notebooks to locate the datasets.

- Permissions: If you encounter permission errors on Ubuntu, ensure the `venv` directory is owned by your current user or use chmod as shown in the setup steps.

- Troubleshooting: If the NLTK downloader is still prompted in the notebook, verify that the `os.environ['NLTK_DATA']` cell in the notebook correctly points to your `venv/nltk_data` path.
